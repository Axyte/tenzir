Get Started

What is Tenzir?

Tenzir is a distributed platform for processing and storing security event data in a pipeline dataflow model.

Dive right in with an interactive tour at tenzir.com and sign up for a free account, or continue below with the open source edition and command line examples.

Install Tenzir​

Select your platform to download and install Tenzir.

  • All Platforms
  • Debian
  • Nix
  • Linux
  • macOS
  • Docker

Use our installer to perform a platform-specific installation:

curl -L get.tenzir.app | sh

The shell script asks you once to confirm the installation.

Download the latest Debian package and install it via dpkg:

dpkg -i tenzir-static-amd64-linux.deb

Try Tenzir with our flake.nix:

nix run github:tenzir/tenzir/stable

Install Tenzir by adding github:tenzir/tenzir/stable to your flake inputs, or use your preferred method to include third-party modules on classic NixOS.

Download a tarball with our static binary for all Linux distributions and unpack it into /opt/tenzir:

tar xzf tenzir-static-x86_64-linux.tar.gz -C /

We also offer prebuilt statically linked binaries for every Git commit to the main branch.

version="$(git describe --abbrev=10 --long --dirty --match='v[0-9]*')"
curl -fsSL "https://storage.googleapis.com/tenzir-dist-public/packages/main/tenzir-${version}-linux-static.tar.gz"

Please use Docker with Rosetta until we offer a native package.

Pull the image:

docker pull tenzir/tenzir

When using Docker, replace tenzir with docker run -it tenzir/tenzir in the examples below.

Download sample data​

The below examples use this dataset. Download to follow along or just keep reading.

# Suricata EVE JSON logs (123 MB)
curl -L -O https://storage.googleapis.com/tenzir-datasets/M57/suricata.tar.gz
tar xzvf suricata.tar.gz
# Zeek TSV logs (43 MB)
curl -L -O https://storage.googleapis.com/tenzir-datasets/M57/zeek.tar.gz
tar xzvf zeek.tar.gz

Run pipelines​

The tenzir exectuable runs a pipeline.

Start with the version source operator and pipe to the write sink operator:

tenzir 'version | write json' 
{"version": "v3.1.0-377-ga790da3049-dirty", "plugins": [{"name": "parquet", "version": "bundled"}, {"name": "pcap", "version": "bundled"}, {"name": "sigma", "version": "bundled"}, {"name": "web", "version": "bundled"}]}

Get the top 3 schemas of the Zeek schemas as tab-separated values (tsv):

cat Zeek/*.log | tenzir '
  read zeek-tsv 
  | measure 
  | summarize events=sum(events) by schema 
  | sort events desc 
  | head 3 
  | write tsv
  '
Output
schema  events
zeek.conn   583838
zeek.ssl    42389
zeek.files  21922

Get the top 5 Zeek notices from notice.log as JSON:

tenzir '
  from file Zeek/notice.log
  | read zeek-tsv 
  | where #schema == "zeek.notice"
  | summarize n=count(msg) by msg
  | sort n desc 
  | head 3 
  | write json
  '
Output
{"msg": "SSL certificate validation failed with (certificate has expired)", "n": 2201}
{"msg": "SSL certificate validation failed with (unable to get local issuer certificate)", "n": 1600}
{"msg": "SSL certificate validation failed with (self signed certificate)", "n": 603}
{"msg": "Detected SMB::FILE_WRITE to admin file share '\\\\10.5.26.4\\C$\\WINDOWS\\h48l10jxplwhq9eowyecjmwg0nxwu72zblns1l3v3c6uu6p6069r4c4c5yjwv_e7.exe'", "n": 339}
{"msg": "SSL certificate validation failed with (certificate is not yet valid)", "n": 324}

Spawn a node​

Use the tenzir-node executable to start a node that manages pipelines and storage.

  • Binary
  • Docker
tenzir-node
[12:43:22.789] node (v3.1.0-377-ga790da3049-dirty) is listening on 127.0.0.1:5158

Expose the port of the listening node and provide a directory for storage:

mkdir storage
docker run -dt -p 5158:5158 -v storage:/var/lib/tenzir tenzir/tenzir --entry-point=tenzir-node

Import data into a node​

End a pipeline with the import operator to ingest data into a node:

tar xOzf zeek.tar.gz | tenzir '
  read zeek
  | import
  '

Filter the input with where to select a subset of events:

tar xOzf suricata.tar.gz | tenzir '
  read suricata
  | where #schema == "suricata.alert"
  | import
  '

Export data from a node​

Start a pipeline with the export operator to initiate a datastream from stored data at a node.

Get a "taste" of one event per schema:

tenzir 'export | taste 1'

zeek.ssl    42389
zeek.files  21922

The above pipeline performs a full scan over the data at the node. Tenzir's pipeline optimizer performs predicate push-down to avoid scans when possible. Consider this pipeline:

tenzir '
  export
  | where *.id.orig_h in 10.0.0.0/8
  | write parquet to file local.parquet
  '

The optimizer coalesces the export and where operators such that expression *.id.orig_h in 10.0.0.0/8 gets pushed down to the index and storage layer.
